marked
Type : Boolean
Default : false

An option when set to true will use marked.js to parse markdown and convert it to HTML. Make sure you have loaded marked.js before loading embed.js if this option is set to true else the plugin will throw an error.

markedOptions
Type : Object
Default : {}

The option takes the marked.js options.

link
Type : Boolean
Default : true

Instructs the plugin whether or not to embed urls/ convert urls into HTML anchor tags.

linkOptions
Type : object
target
Type : String
Default: 'self'

Same as the target attribute in html anchor tag . supports all html supported target values.

exclude
Type : Array
Default : []

Accept array of extensions to be excluded from converting into HTML anchor links

rel
Type : string
Default : ''

Same as the rel attribute.

emoji
Type : Boolean
Default : true

Set to false if you want to disable converting emoji codes into actual emojis.

customEmoji
Type : Array
Default : []

Include the names of custom emojis. Eg : ['emoji1', 'emoji2']. Now they can be used using :emoji1: :emoji2:

fontIcons
Type : Boolean
Default : true

customFontIcons
Type : Array
Default : []

You can use your customised font icon with this plugin. You need to replace the default font files with yours and provide the code to icon mapping in the format : 

customFontIcons:[{
    'text': ' :) ',
    'code': '&#xe60a'
}, {
    'text': ' :D ',
    'code': '&#xe608'
}, {
    'text': ' :d ',
    'code': '&#xe608'
}]

highlightCode
Type : Boolean
Default : true

Set to true if you want to convert code blocks into formatted code. It uses highlight.js for code highlighting. Make sure highlight.js is loaded before embed.js if this option is set to true or else set this to false The format that it supports are single backticks and code blocks: 

```[optional-language-name]
//code here
```

.
videoJS
Type : Boolean
Default : false

Set to true if you want to enable video.js support for audio and video files. Make sure video.js is loaded before the plugin else the plugin will throw an error.

videojsOptions
Type : Object
Default : {}

This object is the video.js options. You can pass any option supported by video.js. Currently it will be applied to both audio and video files. Eg : 

videojsOptions : {
    fluid              : true,
    preload            : 'metadata'
}

Type : Boolean
